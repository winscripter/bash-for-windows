Bash for Windows Docs

Our TikTok

Microsoft Windows has WSL, which is Bash (Linux Terminal, located in C:\Windows\System32\wsl.exe). However, the program takes an incredibly long time to start up.

We had to find a program that does the same operation as Microsoft WSL, but it would take way less time to execute.

So, we have invented a C# program that runs on .NET Framework, and that is Bash for Windows. It starts up in mere 0.3 seconds!

However, this is only version 1.0. There are many commands that are currently missing.


Using Bash for Windows

You can run bash.exe to start up Bash for Windows. A command-prompt style window pops up, that's your program. If the program cannot succeed some type of operation, try running it as administrator, and it should help.

If you need the source code, open SourceCode.txt, or if you have Visual Studio Code installed, you can also run bash.cs.


But how do I test this if I am not on Windows?

We know that running this on Linux makes absolutely no sense. But, even if you want to, or you use other type of OS that also doesn't support EXEs or Bash, here's how:

1. Open online C# Compiler

2. Paste your code into the compiler.

3. Click Execute

*Note that the compiler breaks if you run it again. In this state, please, press Ctrl+F5 and confirm website restart. The difference between CTRL+F5 and F5 is that F5 just reloads the page and loads all the caches, whereas CTRL+F5 not only reloads the page, but also clears all the cache. If a compiler keeps permanently trying to execute the program, close the compiler tab, wait about 5 minutes and run it again.

Avoid program errors

If you're getting some errors in your program, and then Windows Error Report screen pops up, telling bash.exe stopped working,
1. Make sure the program has privileges to the specified object or file
2. Try starting it as administrator
3. Don't rename the EXE file, it must be named as bash.exe, all lowercase letters.

Linux to Batch

[?] The ones that have the star * in them means they can only run in BAT files.

*pwd -> echo %~dp0

ps -> tasklist

rm -f -> del

rm -r -> rmdir / rd

rm -rf -> rmdir /s /q <dirname> / rd /s /q <dirname>

cat -> for /f %%a in ('Filename') do echo %%a

kill 7 / kill 9 -> taskkill /im <process name.exe> /f

mkdir -> mkdir

echo -> echo

touch -> echo. > <Filename>

ls <any arg> -> dir



Compact Experience

The compact experience to this is that, Windows adds the System32 folder to PATH automatically. This means, you can just copy bash.exe and no other file into System32. So next time you can just enter 'bash' and it opens up.



Questions that may be asked

1. Why isn't CD command working?

Because that would require way too much code. Probably version 1.1 or 2.0 may address this bug or feature.


2. Why can't I create a file using echo STRING > FILE?

Check the folder permissions, or run a program as administrator.


3. I executed the program from CMD, but how do I terminate it from CMD?

Apart from closing the command prompt completely, you can simply press Ctrl+C without selecting any text. This will print ^C and terminate the application, but not CMD. You can also kill bash.exe in Task Manager.
